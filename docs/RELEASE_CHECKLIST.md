# 愈后食光 HealMeal 复赛验收清单

> 目标：从 Demo 形态升级为可上线网页版本（Next.js App Router + Supabase 社区后端）。

## 1. 隐私与数据安全

- [ ] 首次建档入口包含隐私用途说明（采集字段、用途、可删除）。
- [ ] 建档流程包含“我已阅读并同意《隐私说明》”勾选，未勾选不可继续。
- [ ] 提供《隐私说明》可读内容（用途、不售卖、不广告、可删除、家属联动可关）。
- [ ] 设置页提供“隐私与数据”入口，支持查看建档只读信息。
- [ ] 支持一键删除（本地数据 + Supabase 关联数据）。
- [ ] 删除后首页回到未建档状态并引导重新建档。

## 2. 餐单体验

- [ ] “生成条件”改为“今天想怎么吃”，弱化工业化表达。
- [ ] 左侧固定标签 + 右侧可点选值（饮食阶段/症状/胃口）。
- [ ] 支持语音输入食材并转 chips，可单个删除。
- [ ] 支持套餐模板：省心版（1菜1汤）/均衡版（2菜1汤+主食）/家常版（3菜1汤）。
- [ ] 生成结果为“菜位数组”，每个菜位支持“换一道”“查看详情”。
- [ ] “烹饪时长”改为“时间预算（可选）”，支持自动估算。

## 3. 打卡闭环

- [ ] 饮食完成度只统计“标记完成”餐次，“采纳”不计入。
- [ ] 记录 `lastCompletedMealKey`，鼓励文案显示最近完成餐次。
- [ ] 运动支持“保存记录”“标记完成”，有目标与达标反馈。
- [ ] 饮水支持目标、快捷加水、记录与达标动作。
- [ ] 情绪与感想可保存并同步到首页摘要。

## 4. 生态社区（Supabase）

- [ ] 使用 Supabase（DB + Auth + Storage）作为社区真实后端。
- [ ] 首次进入生态自动匿名登录（session/cookie 生效）。
- [ ] 帖子、点赞、收藏、评论为真实数据，跨设备可见。
- [ ] 发帖支持文本 + 图片上传（Storage）。
- [ ] 支持复制链接与 Web Share。
- [ ] “我的”页面可查看我发过/我点赞/我收藏。

## 5. 家属页可用性

- [ ] 便捷买菜模块置顶。
- [ ] 阿里入口（淘宝闪购/饿了么）放第一位且最显眼。
- [ ] 所有平台入口可真实跳转，支持 H5 与回退策略。
- [ ] 外链前有轻提示（跳转第三方平台）。

## 6. AI 体验与稳定性

- [ ] 点击后 500ms 内有反馈（骨架屏或流式起始内容）。
- [ ] 支持流式输出（先结论，再补充细节）。
- [ ] 具备 6 小时缓存（同日 + 同阶段 + 同问题上下文命中）。
- [ ] 图片上传前压缩（长边 768~1024）。
- [ ] 错误统一友好提示 + 重试，UI 不出现裸 `not-allowed`。
- [ ] 快慢模型可由环境变量切换，超时分级生效。

## 7. 发布前验证

- [ ] `npm run dev` 可启动。
- [ ] `npm run build` 通过。
- [ ] `npm run start` 可运行。
- [ ] README 已更新环境变量、Supabase 初始化、部署建议（Vercel）。
